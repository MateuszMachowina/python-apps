# ✦ Instagram Local Scripts

A set of Python scripts for downloading, viewing, exporting, and tracking your Instagram data - all running locally on your machine, no third-party services involved.

**Key advantage:** The scripts never ask for your password. They read your active session directly from Firefox's local cookie database, the same way your browser already does.

---

## 📦 Requirements

| Tool | Why it's needed |
|------|----------------|
| **Python 3.11+** | Runtime for all scripts |
| **Mozilla Firefox** | Must be logged into Instagram — scripts read session from it |
| **FFmpeg** | Video processing and thumbnail extraction — add to Windows PATH |
| **VLC Media Player** | Video playback in the viewer — must be 64-bit, matching Python |

### Install Python dependencies

```
pip install requests pillow python-vlc
```

## 📥 `downloader_highlights.py` — Highlights Downloader

Downloads all highlights from any Instagram account you have access to.

**Features:**
- Auto-detects your Firefox session cookie — no password entry
- Supports multiple accounts — each saved to its own subfolder (`highlights/username/`)
- Downloads original quality MP4s and JPEGs at the highest available resolution
- Also downloads the highlight cover image (`cover.jpg`)
- Skips already-downloaded files on re-run — safe to sync repeatedly
- Saves `meta.json` alongside each collection for use by viewer and exporter

**Usage:**
1. Make sure you are logged into Instagram in Firefox
2. Run `python downloader_highlights.py`
3. Enter the target username — session and User ID are auto-filled from Firefox
4. Click **⬇ Sync Highlights**

> **💡 Tip on 429 errors:** Instagram rate-limits repeated requests from the same IP. If you see `ERROR 429`, wait 15–30 minutes before trying again. Your numeric User ID is auto-detected from Firefox cookies (`ds_user_id`) and can be pasted into the User ID field to skip the lookup endpoint that most often triggers the limit.

## 👁️ `viewer_highlights.py` — Highlights Viewer

A local viewer that plays your downloaded highlights with an Instagram-style interface.

**Features:**
- Sidebar with gradient avatar rings, smooth supersampled thumbnails, and cover images
- Story progress bars at the top of the player
- Header row with avatar, collection name, and story counter (`4 / 21`)
- Tap left/right half of the video to go prev/next — just like Instagram
- VLC-powered video playback with original audio
- Country flags downloaded automatically and cached to `flags/` folder
- Adjustable photo display duration (the `+`/`−` control in the top-right corner)
- Supports multiple accounts — grouped by `@username` in the sidebar
- Keyboard shortcuts: `←` `→` to navigate, `Space` to pause, `↑` `↓` to switch collections, `Esc` to close

**Usage:**
```
python viewer_highlights.py
```

Requires highlights to be downloaded first with `downloader_highlights.py`.

## 🎬 `export_highlights.py` — Highlight Exporter

Converts downloaded highlight collections into single MP4 files that look like screen recordings from Instagram.

**Features:**
- **1080×1920** output (Instagram story resolution — matches iPhone screen recording)
- Instagram-style overlay baked into the video:
  - Dark gradient scrim behind the header
  - Story progress bars (white = watched, dimmed = upcoming)
  - Gradient avatar ring (red → pink → purple → gold) with cover thumbnail
  - Collection title, country flag, and story counter
  - Subtle bottom gradient like Instagram's caption area
- Crossfade transitions (~0.3s) between clips via ffmpeg `xfade`
- Original audio preserved from video clips
- Photos displayed for a configurable duration
- Outputs to `exports/<username>/<highlight_title>.mp4`
- Scrollable checklist — pick exactly which highlights to export
- Grouped by `@username` when multiple accounts are present

**Usage:**
```
python export_highlights.py
```

Requires `ffmpeg` on PATH and highlights downloaded first.

## 🕵️ `unfollowers_tracker.py` — Unfollowers Tracker

A console script that tracks changes in your follower/following lists over time and generates a readable report.

**Features:**
- Reads your session automatically from Firefox — no login needed
- Compares current state against the previous saved state
- Generates a `.txt` report with three sections:
  - 💔 **Unfollowers** — people who stopped following you since last run
  - 🎉 **New followers** — people who newly started following you
  - 🔍 **Non-followers** — everyone you follow who doesn't follow you back
- Saves current follower list to `followers_state_<user_id>.json` for next comparison
- Built-in 4-second delay between paginated requests to avoid rate limiting

**Usage:**
```
python unfollowers_tracker.py
```

Run it once to save the baseline. Run it again later to see what changed. The report is saved to `raport_<user_id>.txt`.

> **⚠️ Important:** Do not reduce the built-in sleep delay below 3–4 seconds. Instagram will rate-limit or temporarily block your IP if requests come too fast.

These scripts run entirely on your local machine. They never send your credentials to any external server — they only communicate directly with Instagram using your existing Firefox session.

**Never commit your personal data to version control.** Add this to your `.gitignore`:

## 🛠️ Troubleshooting

**"No Instagram session found in Firefox"**
Open Firefox, go to `instagram.com`, log in, then re-run the script.

**VLC not working / black video screen**
Make sure you have the **64-bit** version of VLC installed matching your Python. Try adding `C:\Program Files\VideoLAN\VLC` to your system PATH.

**FFmpeg not found**
Download from [ffmpeg.org](https://ffmpeg.org/download.html), extract, and add the `bin/` folder to your Windows PATH environment variable. Restart your terminal after.

**Error 429 — rate limited**
Instagram temporarily blocked requests from your IP. Wait 15–30 minutes. In `downloader_highlights.py`, paste your numeric User ID into the User ID field to skip the lookup call.

**Thumbnails missing in viewer sidebar**
FFmpeg is required for extracting video thumbnails. Install it and add to PATH.

**Flags not showing**
The viewer downloads flag PNGs from `flagcdn.com` on first use. Make sure you have an internet connection on first launch. Flags are then cached locally in `flags/` and work offline.
